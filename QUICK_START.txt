PORTFOLIX COMPASS - GEMINI AI/ML QUICK START
========================================================================

⏱️  5 MINUTES TO WORKING AI/ML SYSTEM

STEP 1: Copy API Key Template
  $ cp .env.example.gemini .env

STEP 2: Run Setup Script
  $ bash scripts/setup-aiml.sh

STEP 3: Get & Add Gemini API Key
  1. Visit: https://aistudio.google.com/api-keys
  2. Create project "Portfolix Compass"
  3. Copy API key
  4. Edit .env and paste key:
     GEMINI_API_KEY=your_key_here

STEP 4: Update server.js
  Add this line (around line 28):
  app.use('/api/aiml', require('./routes/aiml.routes'));

STEP 5: Install & Start
  $ npm install @google/generative-ai node-cache
  $ npm run dev

STEP 6: Test
  $ curl http://localhost:5000/api/aiml/health \
    -H "Authorization: Bearer YOUR_TOKEN"

API ENDPOINT EXAMPLES:
========================================================================

1. SALARY PREDICTION
   POST /api/aiml/salary/predict
   Body: { employeeId, currentSalary, experience, performanceRating }

2. OFFER LETTER
   POST /api/aiml/documents/offer-letter
   Body: { name, position, salary, joiningDate }

3. TAX OPTIMIZATION
   POST /api/aiml/tax/optimize-strategy
   Body: { employeeId, annualSalary, age }

4. ANOMALY DETECTION
   POST /api/aiml/payroll/detect-anomalies
   Body: { employees: [...] }

5. MARKET BENCHMARK
   POST /api/aiml/compensation/market-benchmark
   Body: { role, location, experience }

========================================================================
TROUBLESHOOTING:
========================================================================
"GEMINI_API_KEY not found"
  → Check .env exists in project root

"Module not found: @google/generative-ai"
  → Run: npm install @google/generative-ai node-cache

"401 Unauthorized"
  → All endpoints require JWT token in Authorization header

"Cannot find setup script"
  → Ensure you're in project root directory
